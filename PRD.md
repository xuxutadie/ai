# 产品需求文档 (PRD) - 青少年人工智能核心素养练习系统 v2.0

## 1. 产品概述
**产品名称**：青少年人工智能核心素养练习系统 (Youth AI Core Literacy Practice System)
**产品定位**：面向中小学生（小学组、初中组）的人工智能核心素养在线模拟考试与练习平台。
**目标用户**：准备参加人工智能相关竞赛或评测的 1-9 年级学生，以及需要管理学生授权码的教师/机构管理员。
**核心价值**：提供高仿真、强交互的在线答题体验；提供基于授权码的轻量级商业化/权限管理机制。

## 2. 视觉与交互规范 (UI/UX)
本系统采用顶级 SaaS 工具级别的**“Apple 级浅色磨砂玻璃（Light Glassmorphism）”**配合**“深邃科幻蓝 WebGL 噪点渐变”**的混合设计语言。

### 2.1 核心视觉风格
- **全局背景**：基于 `ogl` (WebGL) 渲染的动态噪点流体渐变组件 (`Grainient`)。使用纯粹、深邃的蓝/紫搭配 (`#3b82f6`, `#6366f1`, `#0ea5e9`)，营造未来教育科技感。
- **组件材质**：所有卡片、面板、输入框均采用“磨砂玻璃”质感 (`backdrop-blur-xl`, `bg-white/60` 或 `bg-white/10`)。
- **色彩规范**：
  - 品牌主色：深蓝/靛蓝 (Blue-600, Indigo-600)
  - 成功/答对：翠绿色 (Emerald-500)
  - 警告/错误：红色 (Red-500)
  - 提示/特殊：琥珀色 (Amber-500)
- **字体与排版**：无衬线现代字体，大标题使用黑体 (`font-black`)，辅助文字使用宽字距 (`tracking-widest`)，高对比度设计。

### 2.2 核心动画交互
- **页面转场**：使用 Framer Motion 实现丝滑的 `fade-in`, `slide-in-from-bottom`, `zoom-in` 等物理弹簧动画。
- **Hover 交互**：卡片悬浮时微缩放 (`scale-110` 内部图标) 和上浮。
- **点击反馈**：按钮点击时带有物理回弹缩放 (`active:scale-95`)。

## 3. 核心功能模块

### 3.1 身份授权模块 (Auth)
**功能描述**：用户进入系统前的强制拦截层，通过授权码验证身份和可用次数。
**业务逻辑**：
- 前端通过输入框获取授权码。
- **验证规则**（前端模拟）：
  - `AI5-` 开头：9.9元/5次计费码。
  - `AI1Y-` 开头：39.9元/1年无限次授权码。
  - `admin888`：管理员后门密码（不计入普通授权，触发特殊跳转）。
- **状态持久化**：验证成功后，授权状态（包含剩余次数）需存入 `localStorage`，刷新页面不丢失。
- **UI 交互**：输入错误弹出红色震动提示；输入正确进入组别选择页面。

### 3.2 组别与赛道选择 (Selection)
**功能描述**：用户授权通过后，选择对应的考试难度和赛道。
**业务逻辑**：
- **第一级（组别）**：
  - 小学组 (Primary)：适合 1-6 年级，调用基础题库。
  - 初中组 (Junior)：适合初中生，调用进阶题库。
- **第二级（赛道）**：
  - 赛道一：开发中（点击提示敬请期待）。
  - 赛道二：主要答题区。点击后，系统扣除 1 次剩余能量（如果是 5 次计费码），进入答题界面。
- **UI 交互**：顶部 Navbar 显示当前剩余能量，提供退出登录按钮。页面底部提供进入“后台管理”的入口（带密码验证）。

### 3.3 核心答题模块 (Quiz)
**功能描述**：核心的考试模拟界面。
**业务逻辑**：
- **数据加载**：根据选定的组别（小学/初中），从本地 JSON 文件（如 `p13_questions.json`）中筛选对应的题目数据。
- **题目渲染**：
  - 支持单选题 (`single`)、多选题 (`multiple`)、判断题 (`boolean`)、填空题 (`fill_in_the_blanks`)、简答题 (`short_answer`)。
  - 界面需展示：当前进度（10/43）、倒计时（60分钟）、当前题型（如：单选题）、本题分值（如：2分）。
- **答题交互与自动流转**：
  - 用户点击选项，高亮选中状态。
  - 点击“确认答案”按钮：
    - **答对**：播放 300ms 绿勾动画，**自动跳转**下一题。
    - **答错**：显示红叉，高亮正确答案，停留 **1秒** 后**自动跳转**下一题。
    - **简答题**：点击确认后展开参考答案，不自动跳转，需手动点击“下一题”。
- **交卷**：最后一题答完后，按钮变为“提交试卷”，点击进入结果页。

### 3.4 成绩报告模块 (Result)
**功能描述**：考试结束后的成绩总结与评估。
**业务逻辑**：
- 接收 `Quiz` 模块传递的最终得分和总题数。
- 根据得分率渲染不同的评价等级（如：S级、A级、B级）和鼓励语。
- 提供“重新训练”按钮，点击返回组别选择页面（无需重新输入授权码）。

### 3.5 管理员后台 (Admin Panel)
**功能描述**：教师或代理商用于生成和管理商业授权码的隐秘面板。
**进入条件**：在组别选择页点击“进入后台管理”，输入专属密码（`xxxb520`）后方可进入。
**业务逻辑**：
- **生成授权码**：
  - 点击“9.9元/5次”按钮：生成 `AI5-` + 6位随机大写字母/数字。
  - 点击“39.9元/1年”按钮：生成 `AI1Y-` + 6位随机大写字母/数字。
- **列表管理**：以列表形式展示当前生成的授权码及其类型，支持“一键复制到剪贴板”和“单条删除”。
- **UI 要求**：高对比度的暗色/高亮文字设计（确保在深蓝背景下清晰可见）。

## 4. 技术架构建议 (React/Vite 栈)
如果交由另一个 AI 重新构建此项目，建议采用以下技术栈：
- **框架**: React 18+ (使用 Vite 构建)
- **样式**: Tailwind CSS (核心) + 原生 CSS (`@layer components` 用于定义复杂的玻璃材质 `.glass-panel`, `.glass-card`)
- **路由**: 鉴于项目轻量，可使用状态机 (`currentScreen` state) 控制单页路由，或使用 `react-router-dom`。
- **动画**: `framer-motion` (用于卡片出场、页面切换)、纯 CSS `@keyframes` (用于背景光晕、渐变循环)。
- **背景特效**: `ogl` 库封装 WebGL 着色器，实现 `Grainient` 噪点渐变组件。
- **图标**: `lucide-react`。

## 5. 数据结构约定
**题目数据结构 (`Question` interface)**:
```typescript
interface Question {
  id: string;          // 唯一ID
  grade: string;       // 适用年级段
  group?: 'primary' | 'junior' | 'both'; // 所属组别
  type: 'single' | 'multiple' | 'boolean' | 'fill_in_the_blanks' | 'short_answer';
  question: string;    // 题干
  options?: Record<string, string>; // 选项 (A, B, C, D)
  answer: string | string[]; // 正确答案
  points: number;      // 本题分值
}
```

**授权状态数据结构 (`AuthStatus` interface)**:
```typescript
interface AuthStatus {
  code: string;        // 授权码
  type: 'FREE' | 'PAID_5' | 'UNLIMITED_1Y'; 
  remaining: number;   // 剩余次数
}
```
